# Deck Arcs

This file defines reusable deck arcs for common presentation situations.

These are not rigid formulas. They are composable patterns derived from common presentation guidance: open with context and audience relevance, avoid covering too much, build a clear narrative progression, mix evidence with story, and close with a clear implication or action.

Sources consulted while drafting this file included guidance surfaced from Harvard Business Review, Quartz resource pages, SlideModel articles on openings and outlines, and Presentation Zen references.

## How to use arcs

- choose one primary arc for the whole deck
- optionally combine it with one secondary arc for a section
- do not combine so many arcs that the deck loses coherence
- adapt the arc to the audience, goal, and available time

## Arc 1: Briefing arc

Use for status updates, executive briefings, internal reviews.

Sequence:

1. title / context
2. executive summary
3. current state
4. key findings
5. implications
6. recommendation
7. next steps

## Arc 2: Problem to solution arc

Use for change proposals, improvement cases, operational fixes.

Sequence:

1. hook or framing problem
2. why this matters now
3. evidence of the problem
4. root causes or constraints
5. proposed solution
6. expected benefits
7. implementation steps
8. ask / decision

## Arc 3: Opportunity arc

Use for strategy, innovation, growth, investment themes.

Sequence:

1. market or situational opening
2. unmet need or opening
3. why now
4. strategic option
5. proof points
6. value potential
7. recommended move
8. next actions

## Arc 4: Decision arc

Use when the audience must approve, reject, or choose.

Sequence:

1. decision required
2. context and objective
3. options considered
4. comparison criteria
5. trade-offs and risks
6. recommended option
7. implementation implications
8. decision ask

## Arc 5: Narrative insight arc

Use for keynote-style, thought leadership, or explanatory talks.

Sequence:

1. hook
2. tension / problem
3. insight or reframe
4. evidence or story sequence
5. implication for the audience
6. memorable close

## Arc 6: Case study arc

Use for project retrospectives, client stories, transformation examples.

Sequence:

1. background
2. challenge
3. approach
4. execution highlights
5. result
6. lesson learned
7. transferability / recommendation

## Arc 7: Teaching arc

Use for training, enablement, process explanation, onboarding.

Sequence:

1. objective
2. why this matters
3. core concept
4. step-by-step explanation
5. example or demonstration
6. common mistakes
7. summary checklist
8. action / practice

## Arc 8: Research arc

Use for analytical or evidence-driven presentations.

Sequence:

1. question or hypothesis
2. scope and method
3. key findings
4. supporting evidence
5. interpretation
6. limitations
7. recommendation or implication

## Arc design rules

- start from audience need, not from template novelty
- introduce the main idea early
- keep each slide focused on one job
- use data to support the message
- close with implication, recommendation, or action when appropriate
